# DEC-001 — n8n as Primary Automation Engine

## Context

SendJob requires a workflow automation layer to orchestrate: intake form submissions, Stripe payment processing, Supabase database writes, Twilio SMS dispatch, and Gmail confirmations. A self-hosted solution was preferred to avoid per-execution SaaS pricing and to maintain control over workflow logic.

## Decision

n8n (self-hosted at nexusfound.cloud, v2.6.4) is the primary automation engine for all backend workflow orchestration.

## Impact

- All workflow logic is centralized in n8n
- HTTP Request nodes are used for all external service calls (see DEC-003)
- Self-hosted infrastructure requires SSL cert management at nexusfound.cloud
- Version-specific bugs must be tracked (v2.6.4 Supabase node issues)
- New integrations default to n8n webhook/HTTP pattern
