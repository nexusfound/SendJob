# DEC-002 — Supabase as Database and Auth Layer

## Context

SendJob needs a managed database with REST API access (for n8n HTTP calls), real-time subscriptions (for dispatcher dashboard live updates), and built-in auth (for dispatcher login). A managed solution was chosen to avoid database operations overhead.

## Decision

Supabase (project ID: tmoxppuxrwlabqzfzvin) serves as the primary database, auth provider, and real-time subscription layer.

## Impact

- All data is accessed via Supabase REST API using HTTP Request nodes in n8n
- Supabase always returns arrays — all field access uses `$json[0].fieldname` pattern
- Always use `RETURNING *` on inserts to capture auto-generated IDs in the same call
- RLS policies must be confirmed before assuming open read/write on any table
- Real-time is off by default on new tables — must be explicitly enabled per table
- Customer IDs auto-generated in format `SJ-001` via Supabase sequence
