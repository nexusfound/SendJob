# DEC-012 — Subdomain Identity via Hidden Form Field (Not Host Header)

## Context

n8n needs to know which client a form submission belongs to. The natural approach would be to parse the Host header from the incoming request. However, browsers block Host header overrides, making this unreliable.

## Decision

The client subdomain is passed as a hidden `<input>` field in the intake form body (`subdomain`), not extracted from the Host header.

## Impact

- Every intake form HTML must include: `<input type="hidden" name="subdomain" value="[clientname]">`
- n8n reads `subdomain` from the request body, not headers
- This is reliable across all browsers and fetch() implementations
- The `getSubdomain()` JS function in the Playbook reads `window.location.hostname` as a display fallback only
