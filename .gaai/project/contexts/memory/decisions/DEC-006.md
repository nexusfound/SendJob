# DEC-006 — Twilio A2P 10DLC for SMS — Native n8n Twilio Node

## Context

SendJob uses SMS for dispatcher notifications, tech assignments, and customer confirmations. Twilio was selected for SMS with A2P 10DLC compliance registration. Unlike Supabase (where native nodes are bugged), the native n8n Twilio node works correctly and is the confirmed integration method per CLAUDE.md.

## Decision

Twilio (A2P 10DLC, Low Volume Mixed) is the SMS provider. All Twilio SMS calls in n8n use the **native n8n Twilio node** — NOT HTTP Request. Credential type: Twilio (Account SID + Auth Token).

## Impact

- A2P 10DLC registration is active — carrier compliance is maintained
- All new numbers must be registered before use
- First SMS to any new contact must include opt-out language (`Reply STOP to unsubscribe`)
- Registered number: +13212489804 (primary, acexpress test client)
- Native Twilio node is correct for n8n — do not substitute HTTP Request for SMS
- **This is the OPPOSITE of the Supabase rule** — Supabase uses HTTP Request; Twilio uses native node
