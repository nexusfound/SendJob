# DEC-004 — Netlify for Playbook Static Hosting — Single index.html per Play

## Context

The Nexus Playbook delivers industry-specific demo/deployment sites to clients. Each play needs a customer-facing form, owner dashboard, and tech/field view. Keeping infrastructure simple is a priority for a solo founder. Netlify offers free tier, custom subdomains, and simple CLI deployment.

## Decision

Each Playbook play is a single `index.html` file (all CSS and JS inline, no separate files) deployed to Netlify as a subdomain under `sendjob.app`.

## Impact

- Canonical template: AC demo (Cool Breeze HVAC) — fork this for all new plays
- Deploy method: Netlify CLI preferred (`netlify deploy --prod`); drag-and-drop acceptable
- Views toggled via JS state, not separate pages or routes
- Form submissions POST directly to n8n webhook URL embedded in the HTML
- No build step, no bundler, no framework overhead
- Subdomains provisioned per client: `{clientname}.sendjob.app`
