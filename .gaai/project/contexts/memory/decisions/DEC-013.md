# DEC-013 — Magic Link Auth for Technicians (Token in techs table)

## Context

Technicians need to access their job dashboard (status buttons: En Route / On Site / Complete) without a traditional login. Techs are typically in the field on mobile — a magic link via SMS is the correct UX pattern.

## Decision

Technician dashboard auth uses magic links. n8n generates a unique token scoped per tech (not per job), stores it in `techs.magic_link_token`. URL format: `[client].sendjob.app/tech?token=[token]`

## Impact

- Token expiry is status-based: valid while tech has any active jobs; expires 24 hours after last job marked Complete
- One active token per tech — new assignment invalidates the previous token
- `techs.token_issued_at` tracks when the current token was generated
- This is different from dispatcher auth (Supabase Auth email/password) — techs never use password login
