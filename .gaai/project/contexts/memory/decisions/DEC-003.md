# DEC-003 — HTTP Request Nodes Over Native Supabase Nodes in n8n

## Context

n8n provides native Supabase community nodes as an alternative to raw HTTP Request nodes. During development on n8n v2.6.4, native Supabase nodes were found to have type-handling bugs that cause silent failures and field miscasting.

## Decision

All Supabase interactions in n8n use HTTP Request nodes only. Native Supabase nodes are explicitly prohibited.

## Impact

- Every Supabase call uses: POST/GET/PATCH to `https://{project}.supabase.co/rest/v1/{table}` with `apikey` and `Authorization: Bearer` headers
- This is a hard constraint — must be enforced in every Delivery session
- Applies until the n8n Supabase node bug is confirmed fixed in a future version
- No exceptions: new workflows, bug fixes, and enhancements all follow this rule
