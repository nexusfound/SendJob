# SendJob — Conventions & Proven Patterns

> Load every Delivery session. These are non-negotiable rules learned from production.
> Do not deviate without updating this file first.

---

## n8n — Core Rules

### Supabase Integration
- **Always use HTTP Request nodes** — never native Supabase nodes
- Native Supabase nodes have type-handling bugs in n8n v2.6.4 (will silently fail or miscast fields)
- Insert: add `Prefer: return=representation` header to POST to get the inserted row back
- **n8n 2.x array access:** HTTP Request node returns first element directly — access as `$('Node Name').item.json.id` NOT `$json[0].id`
- Insert pattern: POST to `https://{project}.supabase.co/rest/v1/{table}` with `apikey` and `Authorization: Bearer` headers

### Twilio SMS
- **Use the native n8n Twilio node** — NOT HTTP Request
- Credential type: Twilio (Account SID + Auth Token)
- This is the OPPOSITE of Supabase: Supabase = HTTP Request; Twilio = native node
- Always include opt-out language in first SMS to a new contact (`Reply STOP to unsubscribe`)

### Email (Resend)
- Transactional email uses **Resend** — NOT Gmail
- Sender: `dispatch@sendjob.app`
- Per-client custom sender domain available as upgrade

### Anthropic API Calls
- Call Anthropic API directly via HTTP Request node (not a community node)
- **Base64-encode** payloads that contain special characters to prevent parse errors
- Model: always confirm current model string before hardcoding

### Google Sheets
- Authenticate via **Service Account credentials** — not OAuth
- OAuth causes token expiry issues in unattended automations

### General n8n Rules
- Never assume a node has access to upstream data without explicit wiring
- Use Code nodes to debug — `return [{json: $input.all()}]` to inspect full payload
- When a workflow errors silently, check for type mismatches at the HTTP Request body level
- **Node references:** always `$('Node Name').item.json.field` — never `$json.field` for upstream nodes
- **Code node output:** always `return [{ json: { ... } }]`
- **Expression mode:** must be explicitly toggled on (orange `=` icon) per field — fields default to fixed value mode
- **Subdomain:** always passed as hidden form field in request body (`subdomain`) — NOT Host header

## Supabase — Schema Conventions

### Full Schema

**`clients`**
```
id              uuid PK (auto)
subdomain       text          -- e.g. "acexpress"
business_name   text          -- e.g. "AC Express"
dispatcher_phone text         -- E.164 format: +13215550100
urgency_keywords text         -- comma-separated: "no AC, no heat, flooding"
payment_link    text          -- client's own Square/Stripe/etc URL
twilio_number   text          -- local 10DLC number
active          boolean
```

**`techs`**
```
id              uuid PK (auto)
client_id       uuid FK → clients.id
name            text          -- e.g. "Mike"
phone           text          -- E.164 format
contact_method  text          -- "sms" or "email"
magic_link_token text         -- generated by n8n on assignment
token_issued_at timestamp     -- for expiry logic
```

**`jobs`**
```
id              uuid PK (auto)
client_id       uuid FK → clients.id
tech_id         uuid FK → techs.id (nullable until assigned)
customer_name   text
customer_phone  text
customer_email  text (nullable)
service_address text          -- NOT "address"
service_type    text
issue_description text
is_urgent       boolean       -- set by n8n urgency detection
urgency         text          -- emergency | soon | scheduled
property_type   text          -- residential | condo | commercial | rental
duration        text          -- today | days | week | weeks | months
someone_home    text          -- yes | no_access | arrange
access_code     text
special_instructions text
preferred_date  date          -- null for emergency
preferred_contact text        -- sms | email
status          text          -- New → Assigned → En Route → On Site → Complete
created_at      timestamp (auto)
assigned_at     timestamp (nullable)
completed_at    timestamp (nullable)
```

### Schema Notes

| Table | Key Notes |
|---|---|
| `jobs` | Address column is `service_address` (not `address`). Tech FK: `jobs.tech_id → techs.id` (FK name: `jobs_tech_id_fkey`). Client FK: `jobs.client_id → clients.id` (FK name: `jobs_client_id_fkey`) |
| `techs` | Magic link token stored here, scoped per tech (not per job) |
| `clients` | All workflows look up config here by subdomain — never hardcode client values |

- Always `Prefer: return=representation` on inserts (n8n) to get inserted row back
- RLS enabled on all 3 tables — anon policies: `clients` SELECT; `jobs` SELECT/INSERT/UPDATE; `techs` SELECT/UPDATE
- **Real-time must be enabled on `jobs` table** in Supabase settings — dashboard uses subscriptions, not polling
- **Join pattern (browser JS):** `?select=*,techs(name)` — Supabase returns `job.techs` as `{ name: "..." }` or `null`

## SMS Message Templates

All messages use `[Business Name]` pulled from `clients.business_name` — never hardcode client names.

| Message | Template |
|---|---|
| Customer confirmation | `[Business Name]: We received your service request (Job #[ID]). We'll be in touch shortly.` |
| Dispatcher alert | `⚡ URGENT \| New Job #[ID] — [Customer] \| [Address] \| [Service Type] \| [Issue]` (URGENT prefix only if is_urgent = true) |
| Tech assignment | `[Business Name] - New job assigned: [Customer], [Address], [Phone]. Issue: [Description]. View job: [client].sendjob.app/tech?token=[token]` |
| En Route | `[Business Name]: Your technician is on the way.` |
| On Site | `[Business Name]: Your technician has arrived.` |
| Job Complete | `[Business Name]: Your job is complete. Pay here: [payment_link]` |

**Job # format:** Last 6 chars of UUID, uppercased. e.g. `A1B2C3`. Full UUID stored in DB.

## Playbook — HTML Conventions

- Each play is a **single `index.html`** — intake form + dispatcher dashboard + tech dashboard
- **Canonical template: AC Express (`hvac.sendjob.app`)** — fork this for every new play
- Design system: Barlow Condensed + Barlow fonts, navy dark UI — see `patterns/design-system.md`
- No separate CSS or JS files — all inline
- Views toggled via JS state, not separate pages
- Every intake form includes: `<input type="hidden" name="subdomain" value="[clientname]">`

### Supabase Anon Key — Placeholder Convention

- **Never embed real keys in deliverable HTML files**
- Always use `SUPABASE_ANON_KEY_PLACEHOLDER` as the placeholder value
- Include a comment: "Replace with anon key from Supabase Dashboard → Project Settings → API → anon public"
- Human replaces placeholder before running `netlify deploy --prod`

### Browser JS — Supabase REST Fetch Pattern

Direct `fetch()` from browser HTML. Uses same headers as n8n HTTP Request pattern but in browser context:

```javascript
fetch(`${SUPABASE_URL}/rest/v1/${table}?select=*&order=created_at.asc`, {
  headers: {
    'apikey': SUPABASE_ANON_KEY,
    'Authorization': 'Bearer ' + SUPABASE_ANON_KEY
  }
})
```

- Supabase still returns arrays in browser context — access as `data[0].fieldname`
- Use `?order=field.asc` / `.desc` for sorting; `?select=col1,col2` to limit columns

### CSS Table-to-Card Responsive Pattern

Convert a `<table>` to a vertical card stack on mobile (≤640px) — CSS only, no JavaScript:

```css
@media (max-width: 640px) {
  .jobs-table, .jobs-table thead, .jobs-table tbody,
  .jobs-table th, .jobs-table td, .jobs-table tr { display: block; }
  .jobs-table thead { display: none; }
  .jobs-table tbody td::before { content: attr(data-label); }
}
```

- Requires `data-label="Column Name"` attribute on every `<td>`
- Used by all dispatcher dashboard stories (E02S05–S09)

## What Has Been Ruled Out

- Native n8n Supabase nodes — bugged in v2.6.4
- Gmail for transactional email — replaced by Resend
- HTTP Request for Twilio SMS in n8n — use native Twilio node
- Website builds as a service — low margin, commodity
- Events/bartending vertical — too competitive (CheckCherry at $29/month)
- GoHighLevel — evaluated, not adopted
- Lovable / WordPress — evaluated, not adopted
