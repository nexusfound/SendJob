# Implementation Report — E02S09: Dashboard Real-Time Job Status Updates

## Summary

Added Supabase real-time subscription to `dispatcher-dashboard/index.html`.
The dashboard now updates instantly when any job is inserted or updated in Supabase —
no polling, no `setInterval`. Uses the existing `supabaseClient` instance (shared with E02S08 auth).

| `dispatcher-dashboard/index.html` | Added real-time CSS state, subscription logic, INSERT/UPDATE handlers, status indicator transitions, channel cleanup on logout |

## Changes Made

### 1. CSS — `.status-dot.live` (AC5)

Added new CSS class for the "live" state — pulsing green dot using `var(--success)` (`#00D68F`),
matching the design system specification in the story notes.

### 2. State variable — `realtimeChannel`

Added `var realtimeChannel = null;` alongside existing state variables.
Holds the active Supabase channel reference; null when not subscribed.

### 3. `data-job-id` attribute on `<tr>` in `buildJobRow()` (AC1)

Each `<tr>` now has `data-job-id` set to `job.id`.
Required for `handleUpdate()` to locate existing rows in the DOM by job ID.

### 4. `setLiveState()` (AC5)

Transitions status indicator to pulsing green + appends " · Live" to the current job count label.
Reads row count from the DOM if the label is not yet set. Strips existing " · Live" suffix to
avoid duplication on reconnect.

### 5. `setDisconnectedState()` (AC6)

Transitions status indicator to red dot + "Disconnected" label.
Called on `CHANNEL_ERROR`, `TIMED_OUT`, or `CLOSED` subscription status.
Does NOT clear the table — last known data is preserved.

### 6. `handleInsert(newRow)` (AC2)

Appends a new `<tr>` to `tbodyEl` for the incoming job.
If the table is in empty/error state, transitions to table view first.
Guards against duplicate rows (idempotent).

### 7. `handleUpdate(updatedRow)` (AC1)

Finds the `<tr>` by `data-job-id`. Updates Status badge, Assigned Tech, and Urgency cells
in-place. If row is not in the DOM (edge case), delegates to `handleInsert()`.

### 8. `handleRealtimeChange(payload)` (AC3)

Dispatcher function. Routes `INSERT` to `handleInsert` and `UPDATE` to `handleUpdate`.
`DELETE` is explicitly not handled (out of scope per story).

### 9. `subscribeRealtime()` (AC3, AC5, AC6, AC7)

Creates a named channel `jobs-realtime` on `supabaseClient` (the same client holding the
E02S08 auth session — satisfies AC7). Subscribes to `postgres_changes` for all events on
`public.jobs`. Callback routes subscription status to `setLiveState()` or `setDisconnectedState()`.
Includes comment documenting the Supabase real-time pre-condition (AC4).

### 10. `showAppView()` — wired subscription (AC7)

`subscribeRealtime()` called inside `showAppView()`, which fires after login or session restore.
This ensures the subscription is always established after the auth session is set.

### 11. Logout handler — channel cleanup (AC6)

`realtimeChannel` is removed via `supabaseClient.removeChannel()` before `signOut()`.
Prevents orphaned WebSocket connections.
