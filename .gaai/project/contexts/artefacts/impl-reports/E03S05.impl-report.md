# E03S05 Implementation Report — Tech Dashboard: En Route and On Site Action Buttons

## Summary

Added En Route and On Site action buttons to `tech-dashboard/index.html`. Buttons appear at the bottom of each job card based on the job's current status. Each button sends a PATCH to the Supabase `jobs` table using the existing anon key (no JWT needed — RLS anon UPDATE policy already in place). After a successful PATCH the card updates in place. On failure an inline error is shown and the button is re-enabled.

## Rules Applied

- Supabase PATCH via `fetch()` with anon key — `conventions.md` Browser JS REST Fetch Pattern
- Inline CSS/JS, single `index.html` — Playbook HTML convention
- `Prefer: return=minimal` header on PATCH (no response body needed, reduces payload)
- No JWT — Story note confirms anon RLS UPDATE policy covers `jobs` table
- `encodeURIComponent(jobUuid)` on UUID query param — safe for special characters
- `Object.assign({}, HEADERS, {...})` to extend headers without mutating the shared constant

---

## Architecture Notes

- **Button state management:** `data-job-uuid` on `.job-card-actions` container, `data-next-status` on button. `wireButtonListener` reads both when button is tapped. After `updateCardButtons`, the actions container is replaced with new innerHTML + a fresh `wireButtonListener` call — no stale closures.
- **Error display:** `.card-error` div always rendered inside `.job-card-actions` (hidden via `display:none`), shown by adding `.visible` class on failure.
- **Visual design:** En Route = amber (`--warn` token, rgba(255,184,0,...)), On Site = green (`--success` token, rgba(0,214,143,...)) — matches dispatcher badge colors as specified in Story notes.
