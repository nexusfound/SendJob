# Implementation Report — E03S06: N8N En Route Customer SMS + Status Update

## Summary

Delivered an n8n polling workflow (`E03S06-en-route-customer-sms.json`) and supporting Supabase schema migration (`E03S06-add-en-route-sms-sent.sql`) that:

- Polls every 30 seconds for jobs with `status = 'En Route'` AND `en_route_sms_sent = false`
- For each match: fetches `business_name` from `clients`, builds canonical SMS, sends via native Twilio node
- Marks `en_route_sms_sent = true` after send (or on null-phone skip) — prevents duplicate sends
- Handles `customer_phone = null` gracefully with no error (AC5)
- SMS failure does not revert job status — `continueOnFail: true` on Twilio node (AC6)
- Twilio `From` field uses `FROM_NUMBER_PLACEHOLDER` — human replaces when campaign approved

## Rules Applied

- DEC-003: HTTP Request nodes for all Supabase calls (GET jobs, GET clients, PATCH jobs)
- DEC-006: Native n8n Twilio node for SMS — NOT HTTP Request
- conventions.md Twilio SMS: native Twilio node, `From = clients.twilio_number` (stored; actual campaign number = placeholder)
- conventions.md SMS templates: canonical En Route template used exactly
- n8n 2.x node reference pattern: `$('Node Name').item.json.field` throughout
- Code node output: `return [{ json: { ... } }]` pattern
- SUPABASE_ANON_KEY_PLACEHOLDER convention: all HTTP Request nodes use placeholder
- `FROM_NUMBER_PLACEHOLDER` used for Twilio From field (campaign not yet approved per user context)

## Known Risks / Limitations

- **Twilio From placeholder:** `FROM_NUMBER_PLACEHOLDER` must be replaced with the approved 10DLC number before the workflow is activated. This is an explicit known condition per user instruction — not a delivery gap.
- **PATCH-after-SMS-failure:** If Twilio send fails AND the PATCH also fails (both have `continueOnFail`), the job will be re-polled on the next cycle and another SMS send will be attempted. This is correct behavior — the sentinel's job is to prevent sends, and if the sentinel write fails, a retry is safe (idempotent template).
- **E03S07 mirror:** This workflow is designed to be easily mirrored for E03S07 (On Site SMS) — the `_comment` field documents the three changes needed.
