# Implementation Report: E02S05 — Dispatcher Dashboard Job List View

## Summary

Single `index.html` dispatcher dashboard delivered to `dispatcher-dashboard/index.html`. Fetches all jobs from Supabase `jobs` table via HTTP Request (no SDK), renders them in a responsive table ordered by oldest-first, with empty-state and error-state handling. Deployable to Netlify as-is after replacing the Supabase anon key placeholder.

## Rules Applied

- Supabase integration via HTTP Request (`fetch()` with `apikey` + `Authorization` headers) — not SDK (conventions.md, critical n8n pattern)
- Single `index.html`, all CSS/JS inline — no separate files (conventions.md, Netlify pattern)
- `escapeHtml()` applied to all user-sourced data rendered to DOM — prevents XSS injection
- Supabase anon key left as `SUPABASE_ANON_KEY_PLACEHOLDER` — embedding secrets in a deliverable file is a governance violation; human must substitute before deploy
- Font: IBM Plex Mono + IBM Plex Sans (Google Fonts CDN) — distinctive, not generic AI defaults (frontend-design skill)

## Design Decisions

- **Aesthetic direction:** Industrial/utilitarian — high readability, functional density for an operational tool
- **Color palette:** Off-white background (#F5F4F1), charcoal text, burnt-orange accent (#C45C2D), muted-red urgency (#B33A3A) — matches trades business context
- **Loading state:** Shimmer skeleton rows instead of spinner — perceived performance is faster
- **Mobile layout:** CSS-only card stack (no JS) — reliable cross-browser, no JavaScript dependency for layout

## Risk Notes

- `tech_name` field: Story notes say "tech_name (or join to techs table)". The implementation selects `tech_name` directly from the `jobs` table. If the live Supabase schema stores the tech reference as a foreign key `tech_id` only (no denormalized `tech_name` column), the Assigned Tech column will show blank/Unassigned for all rows. The human must confirm the schema field name before deploy. A PostgREST join query (`?select=...,techs(name)`) would be needed if only `tech_id` is present.
- Supabase anon key: must be replaced before deployment.
- RLS policies: the Story notes that auth is out of scope (E02S08). The anon key read must be allowed by Supabase RLS on the `jobs` table — the human must confirm RLS permits unauthenticated reads, or temporarily disable RLS for testing.
