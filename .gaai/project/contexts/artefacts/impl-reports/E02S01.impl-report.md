## Acceptance Criteria Coverage

### AC1 — Twilio forwards inbound SMS to n8n webhook over HTTPS

**Status: Configured — manual activation required**

The n8n workflow JSON (`E02S01-sendjob-intake-webhook.json`) defines the Webhook node at path `sendjob-intake`, HTTP method POST. When imported and activated in n8n at `nexusfound.cloud`, the Production URL will be `https://nexusfound.cloud/webhook/sendjob-intake`.

Manual step: Log into Twilio Console → Phone Numbers → +13212489804 → Messaging → set webhook URL to `https://nexusfound.cloud/webhook/sendjob-intake`, HTTP method POST, Save.

### AC2 — n8n webhook returns 200 OK within 15 seconds

**Status: Implemented**

The workflow uses `responseMode: "responseNode"` (Respond to Webhook node). This fires immediately on webhook receipt — before downstream processing completes. The `Respond to Webhook` node is connected to Webhook output 0 (top branch), which fires first. Response code is set to 200.

This pattern decouples the 200 ACK from downstream processing duration, guaranteeing Twilio receives 200 OK well within 15 seconds.

### AC3 — SSL certificate on nexusfound.cloud is valid before wiring

**Status: Manual verification required (pre-wiring gate)**

Per execution plan, this must be confirmed in a browser before the Twilio webhook URL is set. Open `https://nexusfound.cloud` in any browser and confirm padlock icon (no certificate warning). If SSL is invalid, stop and escalate — do not wire Twilio.

This is a pre-condition check, not a code change. It cannot be automated from this environment.

### AC4 — Message body and sender phone available as workflow fields

**Status: Implemented**

The `Parse SMS Fields` Code node (position 3 in the workflow) extracts:
- `Body` → `message_body` (dispatcher's SMS reply text)
- `From` → `from_phone` (sender E.164 phone number)
- `To` → `to_phone` (Twilio destination number)

Access pattern in downstream nodes (E02S02+):
```
$('Parse SMS Fields').item.json.message_body   // e.g. "Mike"
$('Parse SMS Fields').item.json.from_phone     // e.g. "+13215550100"
$('Parse SMS Fields').item.json.to_phone       // e.g. "+13212489804"
```

**n8n v2 form-encoded body access:** Twilio sends `application/x-www-form-urlencoded`. The n8n v2 Webhook node exposes this under `$('Webhook').item.json.body.Body` (not `$('Webhook').item.json.Body`). The Code node handles this correctly.

### AC5 — Webhook correctly receives test messages from registered dispatcher phone

**Status: Verifiable via Hoppscotch simulation (live SMS blocked pending A2P)**

A2P 10DLC campaign is pending — live SMS from +13212489804 is blocked. Full AC5 verification is done via Hoppscotch simulation:

```
Method: POST
URL: https://nexusfound.cloud/webhook/sendjob-intake
Headers: Content-Type: application/x-www-form-urlencoded
Body: Body=Tech+Mike&From=%2B13215550100&To=%2B13212489804
Expected: 200 OK
```

This simulates exactly what Twilio POSTs. When A2P is approved, send a real SMS from the dispatcher phone to +13212489804 and confirm it appears in n8n execution history.

### AC6 — Duplicate/retry handling — workflow is idempotent at entry point

**Status: Implemented by design**

E02S01 is the entry-point only — no database writes occur at this layer. The workflow:
1. Receives POST
2. Returns 200 OK immediately (prevents Twilio retry loops)
3. Passes fields to downstream Parse node (async)

If Twilio retries due to a transient failure, the duplicate POST will simply pass through again without side effects at the E02S01 layer. DB-level idempotency (preventing duplicate job assignments) is deferred to E02S02 per the story's Out of Scope definition.

## n8n Wiring Notes (Critical — Post-Import)

After importing `E02S01-sendjob-intake-webhook.json` into n8n:

1. **Verify Webhook → Respond to Webhook connection is on output 0** (top wire). In n8n v2, `responseNode` mode creates TWO outputs from the Webhook node. Output 0 (top) must go to `Respond to Webhook`. Output 1 (bottom) goes to `Parse SMS Fields`. If these are reversed, the 200 OK fires AFTER downstream processing — defeating the 15s guarantee.

2. **Activate the workflow** (toggle at top right of n8n editor). The Production URL only works when the workflow is active.

3. **Copy the Production webhook URL** from the Webhook node's info panel. It should be `https://nexusfound.cloud/webhook/sendjob-intake`. If the URL shown is different, use whatever n8n displays — Twilio must match exactly.

4. **Paste URL into Twilio Console** as described in AC1.

## Known Risks / Limitations

1. **A2P pending:** Live SMS testing is blocked until the Twilio A2P 10DLC campaign is approved. The workflow and webhook wiring can be fully verified via Hoppscotch simulation in the interim.
2. **Response mode wiring:** The Webhook→Respond connection must be on output 0, not output 1. This is a known import issue in n8n v2 (splitInBatches wiring bug is related). Manually verify after import.
3. **No existing Sprint 1 intake workflow found:** The workflow JSON was created fresh. If a `sendjob-intake` workflow already exists in n8n from Sprint 1, compare the Webhook node configuration and update it rather than importing a duplicate.
