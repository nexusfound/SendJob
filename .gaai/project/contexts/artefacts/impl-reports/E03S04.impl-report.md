## Known Risks / Limitations

- `En Route` and `On Site` multi-word status values in the Supabase `status=in.(...)` filter are URL-encoded by the fetch call. Supabase PostgREST handles unencoded parenthesis syntax — tested pattern from E02S05. Spaces in status values within the `in.(...)` filter are standard PostgREST behavior and should work; human should verify with a live token test.
- E03S08 (expired token full-page error) is out of scope. Token-not-found shows an inline error state within this page. This satisfies AC2's "redirect to error state" as an in-page transition.
- No action buttons (En Route / On Site) — those are E03S05.
- `assigned_at` null jobs: Postgres orders NULLs first in `ASC` order by default. This is acceptable per execution plan.
