# Memory Delta — E03S08

## Memory Alignment Check

### New Patterns Introduced

**Split error state pattern (token vs. network):** Rather than a single generic error state, the tech dashboard now differentiates between two distinct error conditions:

- Token missing/invalid → `showTokenError()` — specific "link expired" message
- Network/HTTP failure → `showNetworkError()` — generic "something went wrong" message

This pattern is not yet documented in `conventions.md`. It is a useful convention for any future HTML playbooks that use token-based auth.

### Recommendation for Discovery

Consider adding to `conventions.md` under "HTML Conventions":

```
### Token-Auth Error State Pattern
When a playbook uses token-in-URL auth (no login screen), separate error states by failure type:
- Token missing or not found → show "Link Expired" message with action instruction
- Network/HTTP failure → show "Something Went Wrong" message
Use separate element IDs. Do not use a single generic error div.
```

### Schema / n8n / Conventions Delta

None. No schema changes, no n8n changes, no new conventions that override existing patterns.

### Decisions Worth Extracting

None. No architectural decisions made. The pattern is a UI-level convention reinforcement, not a new technology or service decision.
