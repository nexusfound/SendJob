# Memory Delta — E02S08: Dispatcher Login (Supabase Auth)

## Decisions to Persist

### Auth Pattern for SendJob Dashboard

The dispatcher dashboard uses Supabase Auth JS v2 CDN (`@supabase/supabase-js@2/dist/umd/supabase.min.js`) for email/password authentication in a single `index.html` file.

**Session pattern:**
- `supabaseClient.auth.getSession()` on `DOMContentLoaded` — restores session from `localStorage` (Supabase v2 default, no additional config needed)
- `supabaseClient.auth.signInWithPassword({email, password})` on form submit
- `supabaseClient.auth.signOut()` on logout

**Authorization pattern:**
- All Supabase REST fetch calls use `Authorization: Bearer ${currentAccessToken || SUPABASE_ANON_KEY}`
- `currentAccessToken` stores `session.access_token` on login/session restore; cleared on logout
- This ensures RLS policies for `authenticated` role evaluate correctly

**Mobile auth form pattern (Playbook design system):**
- Inputs: `font-size: 16px` (prevents iOS auto-zoom), `min-height: var(--tap-min)` (52px)
- Focus ring: `border-color: var(--ice)`
- Card: `--navy-mid` background, `--border` border

## No Architecture Changes

- Single file modification pattern confirmed appropriate for auth gate in this stack
- No new files, no schema changes, no new dependencies beyond Supabase CDN (already required for Auth)
