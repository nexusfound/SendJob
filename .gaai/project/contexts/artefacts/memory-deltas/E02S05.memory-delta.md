### 1. CSS Table-to-Card-Stack Responsive Pattern

**Pattern:** Convert a `<table>` to a vertical card stack on mobile using CSS only (no JavaScript).

**Implementation:**
```css
@media (max-width: 640px) {
  .jobs-table, .jobs-table thead, .jobs-table tbody,
  .jobs-table th, .jobs-table td, .jobs-table tr { display: block; }
  .jobs-table thead { display: none; }
  .jobs-table tbody td::before { content: attr(data-label); /* inject label */ }
}
```

Requires `data-label="Column Name"` on every `<td>`.

**Signal:** This pattern will be needed by E02S06 (urgency badges), E02S07 (assignment dropdown), E02S08 (auth overlay), E02S09 (real-time). Recommend persisting now.

### 2. Supabase REST Fetch (Browser JS)

**Pattern:** Direct `fetch()` from browser HTML using Supabase REST API with anon key headers.

**Note:** conventions.md covers the n8n HTTP Request node pattern. The browser-side `fetch()` pattern uses the same headers but different context.

```javascript
fetch(`${SUPABASE_URL}/rest/v1/${table}?select=...&order=field.asc`, {
  headers: {
    'apikey': SUPABASE_ANON_KEY,
    'Authorization': 'Bearer ' + SUPABASE_ANON_KEY
  }
})
```

**Signal:** E02S07 (PATCH for assignment), E02S08 (auth), and E02S09 (real-time) will all need browser-side Supabase patterns. Recommend adding browser-fetch section to conventions.md.

---

### 3. Supabase Anon Key Placeholder Convention

**Pattern:** Always use `SUPABASE_ANON_KEY_PLACEHOLDER` as the key value in deliverable HTML files. Never embed real keys. Include a comment telling the human where to find the real key.

**Signal:** Will recur on every Playbook HTML file. Should be explicit convention.
