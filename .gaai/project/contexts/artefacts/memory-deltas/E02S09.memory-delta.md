# Memory Delta — E02S09: Dashboard Real-Time Job Status Updates

## Patterns Worth Persisting

### Supabase Real-Time — Browser JS Pattern

Proven working pattern for Supabase real-time subscriptions in vanilla-JS dashboard:

```javascript
// Use the same supabaseClient instance as auth — never create a second client
var realtimeChannel = supabaseClient
  .channel('jobs-realtime')
  .on('postgres_changes', { event: '*', schema: 'public', table: 'jobs' }, handleChange)
  .subscribe(function(status) {
    if (status === 'SUBSCRIBED') { /* show live indicator */ }
    if (status === 'CHANNEL_ERROR' || status === 'TIMED_OUT') { /* show disconnected */ }
  });

// Cleanup on logout
supabaseClient.removeChannel(realtimeChannel);
```

**Pre-condition (always required):** Enable real-time on the table in Supabase Dashboard →
Database → Replication. Off by default on new tables. Failure to do this = subscription
appears to work but no events fire.

### In-Place Row Update Pattern

For real-time UPDATE events in a `<table>`, use `data-job-id` on `<tr>` for efficient row lookup:

```javascript
tr.setAttribute('data-job-id', job.id);
// ...
var tr = tbodyEl.querySelector('tr[data-job-id="' + rowId + '"]');
```

Update individual cells via `td.innerHTML = ...` rather than re-rendering the full row.
This preserves dropdown state and avoids re-attaching event listeners.

### Status Indicator States (Dashboard Design System)

Full lifecycle for dashboard connection status:

| State | CSS class | Color | Animation | Label |
|---|---|---|---|---|
| Loading | `.loading` | `--ice` (#00C8FF) | pulse | "Loading…" |
| Loaded | `.loaded` | `--success` (#00D68F) | none | "{N} jobs" |
| Live | `.live` | `--success` (#00D68F) | pulse | "{N} jobs · Live" |
| Error/Disconnected | `.error` | `--urgent` (#FF4444) | none | "Error" / "Disconnected" |
