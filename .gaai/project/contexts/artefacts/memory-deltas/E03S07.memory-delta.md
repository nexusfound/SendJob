# Memory Delta — E03S07: N8N On Site Customer SMS + Status Update

## New Patterns / Learnings

### Pattern: Polling + Sentinel for Status-Triggered Customer SMS (Confirmed)

E03S07 confirms the polling + sentinel pattern established in E03S06 (DEC-016) works cleanly for multiple status transitions. The pattern is now proven for both `En Route` and `On Site` transitions. It can be applied directly to any future status-triggered SMS (e.g., Job Complete in Sprint 4) without architectural evaluation.

**Pattern summary:**
- Schedule Trigger (30s) → GET jobs where `status=eq.[STATUS]&[sentinel]=eq.false`
- Split In Items → GET Client Details → Customer Phone Present? (IF) → Build SMS Body → Send Customer SMS (native Twilio, continueOnFail) → PATCH sentinel=true (continueOnFail)
- Three variables to change per new status: filter value, sentinel column name, SMS template

### Confirmed: No New RLS Policy for Subsequent Polling Workflows

The `jobs` table anon UPDATE policy (confirmed E03S05, validated E03S06) covers all polling workflow sentinel writes. Future On Site / Job Complete SMS workflows do not need new RLS policies.

### Import Warning: Split In Items Loop/Done Rewiring

Confirmed recurring issue: n8n v2.6.4 JSON import always wires Split In Items to `done` output. Must manually rewire to `loop` output on every workflow import. This is documented in E03S06 (DEC-016) and applies to all future polling workflows with Split In Items.

## No New Decisions Required

DEC-016 (polling + sentinel pattern, documented in E03S06) covers this delivery. No new DEC entry needed — pattern is already recorded.
