# E03S03 Memory Delta — N8N: Tech Notification SMS with Magic Link

## Decisions / Patterns Worth Persisting

### 1. E03S03 — Webhook-to-Webhook Chaining Pattern for n8n Workflow Sequences

**Context:** E03S02 (token generation) and E03S03 (tech notification) are separate workflows. E03S03 is triggered by a POST webhook call from E03S02's success branch.

**Pattern:** When n8n workflows must sequence (A then B), use an HTTP Request node in workflow A to call the webhook trigger of workflow B. Pass all required context in the POST body. This avoids modifying delivered workflows and maintains clean separation.

**Integration body pattern:**
```json
{
  "tech_id": "{{ $('Generate Magic Link Token').item.json.tech_id }}",
  "job_id": "{{ $('Trigger: Job Assigned Webhook').item.json.body.job_id }}",
  "token": "{{ $('Generate Magic Link Token').item.json.token }}",
  "client_id": "{{ $('Trigger: Job Assigned Webhook').item.json.body.client_id }}"
}
```

**File:** `n8n-workflows/E03S03-tech-notification-sms.json`, `_comment` field

---

### 2. Opt-Out Language — Always Append (Not First-SMS-Only)

**Context:** Story notes said "include opt-out on first SMS to new contact." Determining "first SMS" requires state tracking not yet implemented.

**Decision:** Always append `Reply STOP to unsubscribe.` to every tech notification SMS. This is safe (over-inclusive), maintains A2P 10DLC compliance, and avoids a state-tracking requirement. Revisit if message length becomes a concern.

---

### 3. Canonical Tech Notification SMS Template (confirmed)

**Confirmed pattern in conventions.md:**
```
[Business Name] - New job assigned: [Customer], [Address], [Phone]. Issue: [Description]. View job: [URL] Reply STOP to unsubscribe.
```

URL format: `https://tech.sendjob.app?token=[token]`

This template is now implemented and QA-verified in `E03S03-tech-notification-sms.json`.
