# Memory Delta — E03S02

## Durable Patterns Identified

### Pattern: Browser-Side Magic Link Token Generation

**Context:** When a browser-only assignment path (no n8n) needs to trigger token generation, use `crypto.randomUUID()` natively in browser JS with the same `fetch()` PATCH pattern as other Supabase writes.

**Code pattern:**
```javascript
function generateMagicLinkToken(techId) {
  if (!techId) { return; }
  var token = crypto.randomUUID();
  fetch(SUPABASE_URL + '/rest/v1/techs?id=eq.' + encodeURIComponent(techId), {
    method: 'PATCH',
    headers: {
      'apikey': SUPABASE_ANON_KEY,
      'Authorization': 'Bearer ' + (currentAccessToken || SUPABASE_ANON_KEY),
      'Content-Type': 'application/json',
      'Prefer': 'return=representation'
    },
    body: JSON.stringify({ magic_link_token: token, token_issued_at: new Date().toISOString() })
  })
  .then(function(r) { if (!r.ok) { console.warn('[SendJob] Token PATCH failed (non-fatal):', r.status); } })
  .catch(function(err) { console.warn('[SendJob] Token PATCH error (non-fatal):', err.message); });
}
```

Fire-and-forget. Never surfaces errors to the user. PATCH failure logged, assignment stands.

### Pattern: n8n Token Generation Node Sequence

**Use when:** n8n workflow needs to generate a magic link token after assignment.

Nodes (in order):
1. Code node: `require('crypto').randomUUID()` → `return [{ json: { tech_id: techId, token: token } }]`
2. HTTP Request PATCH: `techs?id=eq.[tech_id]`, body `{ magic_link_token, token_issued_at }`, `continueOnFail: true`
3. IF node: check `$('PATCH node').item.json.id` is not empty (return=representation confirms write)
4. False branch: Code node logs error only — never modifies jobs table

**Key rule:** `continueOnFail: true` on the PATCH node is mandatory — AC5 requires token failure to be isolated from job assignment.
