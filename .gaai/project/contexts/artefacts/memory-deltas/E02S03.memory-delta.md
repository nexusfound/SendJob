# Memory Delta — E02S03: Error SMS if Tech Name Not Recognized

## New Patterns to Persist

### Pattern: $input.all() for multi-row Supabase array response in Code node

When a Supabase GET returns multiple rows (e.g., fetching all techs for a client), and a
downstream Code node needs to iterate all rows, use `$input.all()`:

```javascript
const allItems = $input.all();
const names = allItems.map(item => item.json.name).filter(Boolean).join(', ');
```

This is the correct n8n v2 pattern for multi-item iteration in Code nodes.
`$json` only accesses the first item in the set — use `$input.all()` for the full array.

### Pattern: n8n IF node FALSE branch — always verify wiring after import

The n8n v2 IF node has two outputs: index 0 (TRUE) and index 1 (FALSE).
After JSON import, n8n has been observed (in v2.6.4) to rewire FALSE branches to the TRUE
output in some cases. Always manually verify FALSE branch connections post-import.

This applies to any IF node with a FALSE branch connection — not just splitInBatches.
