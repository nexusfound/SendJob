## Delta 1 — n8n v2 Form-Encoded Body Access (ADD to conventions.md)

**File:** `contexts/memory/patterns/conventions.md`
**Section:** n8n — Core Rules → General n8n Rules

**New pattern confirmed by E02S01 implementation:**

Twilio (and any service that POSTs `application/x-www-form-urlencoded`) has its body parsed by the n8n v2 Webhook node and exposed under a `.body` sub-key:

```
$('Webhook').item.json.body.Body    // Twilio SMS text
$('Webhook').item.json.body.From    // Sender phone (E.164)
$('Webhook').item.json.body.To      // Twilio destination number
```

This is distinct from JSON body POSTs, where fields are at the top level.

**Recommended addition to conventions.md:**
> **Webhook body access (form-encoded):** For services POSTing `application/x-www-form-urlencoded` (e.g. Twilio inbound SMS), the n8n v2 Webhook node wraps fields under `.body`: `$('Webhook').item.json.body.FieldName`. JSON POST bodies are at the top level: `$('Webhook').item.json.fieldName`.

## Delta 2 — Webhook responseMode for Twilio (CONFIRM existing pattern)

**File:** `contexts/memory/patterns/conventions.md`

The `responseMode: "responseNode"` + `Respond to Webhook` node pattern for guaranteeing immediate 200 OK is a standard n8n pattern. It was not previously documented explicitly in conventions.md.

**Recommended addition to conventions.md (n8n Core Rules):**
> **Webhook response mode (Twilio ACK):** Always use `responseMode: "responseNode"` on Webhook nodes that receive Twilio callbacks. Wire `Respond to Webhook` (200 OK) to Webhook output 0. Wire downstream processing to Webhook output 1. This ensures Twilio receives 200 OK immediately and does not trigger retry loops.
