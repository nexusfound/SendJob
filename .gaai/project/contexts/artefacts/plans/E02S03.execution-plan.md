# Execution Plan — E02S03: Error SMS if Tech Name Not Recognized

## Tier Assessment

**Tier 1 — MicroDelivery**
- Complexity: 1
- Files affected: 1 (n8n-workflows/sendjob-intake-combined.json)
- AC count: 5
- Specialists: none

## Approach

Extend the existing `sendjob-intake-combined.json` workflow by adding the FALSE branch
of the `Tech Found?` IF node. No new workflow file needed — this is an in-place extension
of the combined E02S01+S02 workflow, which explicitly documented this hook point.

## Steps

1. Add `Fetch All Techs` HTTP Request node — GET `/rest/v1/techs?client_id=eq.{id}&select=name`
   — retrieves all tech names live for the client (AC5)
2. Add `Build Error SMS` Code node — constructs the error message using the dispatcher's
   attempted name and the live tech list (AC2)
3. Add `Send Error SMS` Twilio node — sends from `clients.twilio_number` to `dispatcher_phone` (AC1, AC3)
4. Wire `Tech Found?` output index 1 (FALSE) → `Fetch All Techs` → `Build Error SMS` → `Send Error SMS`
5. Verify no PATCH node in the FALSE path (AC4 — job status unchanged)

## Constraints Applied

- HTTP Request nodes for Supabase (conventions.md)
- Native Twilio node for SMS (story notes explicit — NOT HTTP Request)
- n8n v2 expressions: `$('NodeName').item.json.field`
- Phone passed directly to Twilio node (no URL encoding needed in node params)
- SMS message under 160 chars for typical tech roster sizes
