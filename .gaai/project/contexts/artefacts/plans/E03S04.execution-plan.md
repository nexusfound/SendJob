# E03S04 Execution Plan — Technician Dashboard (Magic Link Auth)

## Summary

Produce a single `tech-dashboard/index.html` deployed to Netlify at `techhub.sendjob.app`.
The page reads `?token=` from the URL, validates it against `techs.magic_link_token` in Supabase
(anon REST), fetches active jobs for that tech, and renders them as mobile-first job cards.
No login screen — token is the only auth mechanism.

**Override applied:** All references use `techhub.sendjob.app` (not `tech.sendjob.app`).

## Implementation Sequence

### Step 1 — HTML shell + design tokens

Create `tech-dashboard/index.html` with:
- `<!DOCTYPE html>`, charset, viewport meta, no-scale
- `theme-color: #0A1628`
- Google Fonts: Barlow Condensed + Barlow
- All CSS design tokens from dispatcher dashboard (exact token set — same design system)
- Body styles, reset

### Step 2 — Supabase config block

At top of `<script>`:
```js
const SUPABASE_URL = 'https://tmoxppuxrwlabqzfzvin.supabase.co';
// Replace with anon key from Supabase Dashboard → Project Settings → API → anon public
const SUPABASE_ANON_KEY = 'SUPABASE_ANON_KEY_PLACEHOLDER';
```

### Step 3 — Token extraction (AC1)

```js
const params = new URLSearchParams(window.location.search);
const token = params.get('token');
if (!token) { showError(); return; }
```

If no token present → show error state immediately (covers AC2 / AC8 error path).

### Step 4 — Token validation (AC2)

```js
const res = await fetch(
  `${SUPABASE_URL}/rest/v1/techs?magic_link_token=eq.${encodeURIComponent(token)}&select=id,name,client_id`,
  { headers: { 'apikey': SUPABASE_ANON_KEY, 'Authorization': 'Bearer ' + SUPABASE_ANON_KEY } }
);
const techs = await res.json();
if (!techs.length) { showError(); return; }
const tech = techs[0];
```

If no match → show error state (redirect to error state per AC2 — inline error section, E03S08 out of scope).

### Step 5 — Jobs fetch (AC3, AC5)

```js
const jobsRes = await fetch(
  `${SUPABASE_URL}/rest/v1/jobs?tech_id=eq.${tech.id}&status=in.(Assigned,En Route,On Site)&order=assigned_at.asc&select=id,customer_name,service_address,service_type,issue_description,status,assigned_at`,
  { headers: { 'apikey': SUPABASE_ANON_KEY, 'Authorization': 'Bearer ' + SUPABASE_ANON_KEY } }
);
const jobs = await jobsRes.json();
```

### Step 6 — Render job cards (AC4, AC5, AC6, AC7)

For each job, render a card:
- Job ID: last 6 chars of UUID uppercased (AC4)
- Customer Name (AC4)
- Service Address (AC4)
- Service Type (AC4)
- Issue Description (AC4)
- Current Status badge (AC4)

If `jobs.length === 0` → render empty state: "No active jobs right now." (AC7)

### Step 7 — Status badge styles

Status badges use color-coded pills:
- `Assigned` → ice blue (`--ice`)
- `En Route` → warm yellow (`--warn`)
- `On Site` → success green (`--success`)

### Step 8 — Mobile card layout (AC6)

Card-based layout — one job per card, full width, stacked vertically.
Each field on its own row with label + value.
Min tap target: all interactive elements 48px height (none in this story — buttons are E03S05).
No table used — pure div/card structure.
Max-width 640px container centered on desktop; full-width on mobile.
No horizontal scroll on 375px viewport enforced by `overflow-x: hidden` on body.

### Step 9 — Loading and error states

- Loading state: spinner or "Loading your jobs..." shown during async fetch
- Error state section: shown if token missing or not found in Supabase
  - Message: "This link has expired or is invalid."
  - Sub-message: "Text JOBS to your dispatch number to get a new link."
  - (Full E03S08 error page is out of scope — a clean error state within this page satisfies AC2's "redirect to error state" requirement)

### Step 10 — Nav header

Sticky nav matching dispatcher dashboard design:
- Left: SendJob logo mark (⚡) + "SENDJOB TECH" in Barlow Condensed
- Right: Status pill showing "LIVE" / "LOADING" / "ERROR"
- No client logo (E03S01 out of scope)

## Risk Register

| Risk | Likelihood | Mitigation |
|------|-----------|------------|

| Mobile viewport overflow on description text | Medium | CSS `word-break: break-word` on description field |
