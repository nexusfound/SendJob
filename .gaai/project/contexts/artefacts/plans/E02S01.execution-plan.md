# Execution Plan — E02S01: Twilio Inbound Webhook

## Summary

Wire Twilio number +13212489804 to forward inbound SMS to the n8n webhook at
`https://nexusfound.cloud/webhook/sendjob-intake`. Build the n8n Webhook node to
receive the POST, expose `Body` and `From` fields to downstream nodes, and return
200 OK within 15 seconds.

No tech matching, no job updates — entry point only.

---

## Approach Evaluation

**Skipped.** All patterns are established in `conventions.md`:
- Twilio inbound SMS → webhook URL (standard Twilio console config)
- n8n Webhook node (standard HTTP trigger, POST method)
- No new libraries or services introduced

---

## Implementation Sequence

### Phase 1 — SSL Verification (AC3)

**Step 1.1 — Verify SSL cert on nexusfound.cloud**

Manual pre-check (browser or Hoppscotch):
- Open `https://nexusfound.cloud` in browser — confirm padlock, no cert warning
- No curl available on Windows; use browser address bar SSL indicator
- If cert is invalid/expired: STOP and escalate to human. Do not proceed to wiring.

Expected result: Valid TLS, padlock shown, no browser warning.

---

### Phase 2 — n8n Webhook Node (AC2, AC4, AC6)

**Step 2.1 — Create or verify the Webhook node in the existing intake workflow**

Target workflow: `sendjob-intake` (already exists from Sprint 1)

Check if the webhook node at `/webhook/sendjob-intake` already exists and is active:
- In n8n, open the `sendjob-intake` workflow
- Confirm Webhook node is present, HTTP method = POST, path = `sendjob-intake`
- If it already exists: note the Production URL and proceed to Phase 3
- If it does not exist: create it (see Step 2.2)

**Step 2.2 — Webhook node configuration (if creation needed)**

Node type: Webhook
- HTTP Method: POST
- Path: `sendjob-intake`
- Response Mode: `Respond to Webhook` (not "Last Node" — ensures 200 OK fires immediately, not after full workflow execution)
- Response Code: 200
- Response Body: (empty or `{"ok":true}`)

This ensures Twilio gets 200 OK immediately, within 15 seconds (AC2), without waiting for downstream processing.

**Step 2.3 — Verify incoming fields are accessible (AC4)**

Twilio POSTs `application/x-www-form-urlencoded` with fields:
- `Body` — the SMS text
- `From` — sender phone number (E.164: +1XXXXXXXXXX)
- `To` — Twilio number

In n8n, the Webhook node parses form-encoded POST bodies automatically. These fields are
accessible as: `$('Webhook').item.json.body.Body`, `$('Webhook').item.json.body.From`

(Note: n8n v2 Webhook node wraps form-encoded body under `.body` sub-key.)

**Step 2.4 — Idempotency / duplicate handling (AC6)**

The entry-point node (this story) does not write to any database. Since it only receives
and routes, it is inherently idempotent at this stage — duplicate POSTs from Twilio retries
will simply pass through the webhook again without side effects at the E02S01 layer.
Downstream deduplication (E02S02+) will handle DB-level idempotency.
Document this in the impl-report for the QA agent.

---

### Phase 3 — Twilio Console Wiring (AC1, AC5)

**Step 3.1 — Set webhook URL in Twilio console**

1. Log into Twilio Console → Phone Numbers → Manage → +13212489804
2. Under "Messaging" → "A message comes in":
   - Webhook: `https://nexusfound.cloud/webhook/sendjob-intake`
   - HTTP Method: POST
3. Save the number configuration

**Step 3.2 — Test with Hoppscotch (AC5 — pre-A2P verification)**

Since A2P campaign is pending (live SMS blocked), simulate Twilio's POST manually:

Using Hoppscotch (hoppscotch.io):
- Method: POST
- URL: `https://nexusfound.cloud/webhook/sendjob-intake`
- Headers: `Content-Type: application/x-www-form-urlencoded`
- Body (raw, URL-encoded): `Body=Tech+Mike&From=%2B13215550100&To=%2B13212489804`
- Expected response: 200 OK (with any body or empty body)

This validates AC1 (webhook reachable), AC2 (200 OK), AC4 (fields present), AC5 (simulated).

## Edge Cases

| AC | Edge Case | Handling |
|----|-----------|----------|
| AC2 | n8n workflow has long-running downstream nodes | Use "Respond to Webhook" mode — fires 200 immediately, decoupled from downstream |
| AC3 | SSL cert expired or self-signed | STOP — do not wire Twilio until cert is valid. Escalate to human. |
| AC4 | Twilio body fields not accessible in n8n | Check n8n Webhook node "Body Parameters" output; may need `$json.body.Body` vs `$json.Body` depending on n8n version |
| AC5 | A2P pending — can't send live SMS | Use Hoppscotch to simulate Twilio POST; documents real test is blocked pending campaign approval |
| AC6 | Twilio retries on timeout | Response Mode = "Respond to Webhook" prevents this by returning immediately |
