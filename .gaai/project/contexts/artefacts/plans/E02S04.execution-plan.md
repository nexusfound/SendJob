# Execution Plan — E02S04: Already-Assigned Guard SMS

## Tier Assessment

**Tier 2 — Core Team**
- Complexity: 2 (3 new nodes added to an existing workflow's false branch)
- Files affected: 1 (`n8n-workflows/sendjob-intake-combined.json`)
- AC count: 5 (exceeds Tier 1 threshold of ≤ 3)
- Specialists triggered: none

## Approach

E02S04 extends the existing `sendjob-intake-combined.json` (E02S01+E02S02+E02S03) workflow.
The false branch of the `Job is New?` IF node is the defined hook point (noted in E02S02 impl-report).

When `Job is New?` fires FALSE, no New job was found (the E02S02 query filters `status=eq.New`).
The most recent job for this client has some other status (Assigned, En Route, On Site, Complete).
Three nodes are needed on the false branch:

1. **Get Current Job State** — HTTP GET to Supabase `jobs` table with Supabase foreign key join
   (`?select=id,status,tech_id,techs(name)`) to retrieve the actual job state and tech name
   in a single call. No `status=eq.New` filter — fetches most recent job regardless of status.

2. **Build Guard SMS** — Code node that assembles the guard message:
   - With tech: `Job #[ID] is already [status]. Assigned to [Tech Name].`
   - Without tech (AC4): `Job #[ID] is already [status].`
   - Edge case (no jobs): fallback message

3. **Send Guard SMS** — Native Twilio node (per conventions.md: Twilio = native node, not HTTP Request).
   Sends from `clients.twilio_number` (AC5) to dispatcher's `from_phone`.

## Conventions Applied

- HTTP Request nodes only for Supabase (conventions.md)
- Native Twilio node for SMS (conventions.md)
- n8n v2 expression pattern: `$('Node Name').item.json.field` throughout
- Code node output: `return [{ json: { ... } }]`
- `SUPABASE_ANON_KEY_PLACEHOLDER` in new HTTP Request node
- Job # format: `.slice(-6).toUpperCase()` per conventions.md

## Risk: Supabase join on false branch

The `?select=techs(name)` join relies on the `jobs_tech_id_fkey` foreign key (confirmed in conventions.md).
When `tech_id` is NULL, Supabase returns `"techs": null` — the Code node handles this gracefully (AC4).
No schema changes required.
