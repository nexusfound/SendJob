# Story E02S09: Dashboard ‚Äî Real-Time Job Status Updates

## User Story

**As a** dispatcher,
**I want** the job list to update automatically when any job status changes,
**so that** I always see live data without manually refreshing the page.

---

## Context

The current dashboard fetches jobs once on page load. When a tech is assigned (via SMS or dashboard dropdown), the change only appears after a manual refresh. Real-time subscriptions on the Supabase `jobs` table eliminate this ‚Äî the dispatcher sees updates the moment they happen.

---

## Acceptance Criteria

- [ ] **AC1:** When a job's `status` or `tech_id` changes in Supabase (from any source ‚Äî SMS, dashboard, or direct DB update), the corresponding row in the dashboard updates within 2 seconds without a page refresh.
- [ ] **AC2:** When a new job is inserted into the `jobs` table, it appears in the dashboard list automatically (oldest-first order maintained).
- [ ] **AC3:** The implementation uses Supabase real-time JS client subscriptions ‚Äî no polling, no `setInterval`.
- [ ] **AC4:** Real-time is enabled on the `jobs` table in Supabase Dashboard settings before this is deployed ‚Äî this is a required pre-condition (off by default on new tables).
- [ ] **AC5:** The status indicator in the nav shows a distinct "live" state (e.g., pulsing green dot) when the real-time subscription is active and connected.
- [ ] **AC6:** If the real-time subscription drops or errors, the dashboard falls back gracefully ‚Äî shows the last known data, updates the status indicator to reflect the disconnected state, and does not crash.
- [ ] **AC7:** Real-time updates work correctly for an authenticated dispatcher (the subscription must respect the Supabase Auth session from E02S08).

---

## Out of Scope

- Real-time on tables other than `jobs`
- Push notifications or browser alerts for new jobs
- Multi-tab sync

---

## Dependencies

- E02S05 ‚Äî job table must be rendered.
- E02S08 ‚Äî auth session must exist; real-time subscription must be established after login.

---

## Notes

- **Pre-condition:** Enable real-time on `jobs` table: Supabase Dashboard ‚Üí Database ‚Üí Replication ‚Üí enable `jobs` table. Do this before deploying.
- Supabase real-time pattern:
  ```javascript
  supabase.channel('jobs')
    .on('postgres_changes', { event: '*', schema: 'public', table: 'jobs' }, handleChange)
    .subscribe();
  ```
- `handleChange` receives `{ eventType, new: rowData, old: rowData }` ‚Äî use `eventType` (`INSERT`, `UPDATE`, `DELETE`) to update the table without a full re-fetch.
- Use the same Supabase JS client instance as E02S08 auth ‚Äî do not create a second client.
- Status indicator states: loading (pulsing blue) ‚Üí loaded (green) ‚Üí live (pulsing green) ‚Üí error (red).
- Design system: "live" state uses `--success: #00D68F`.
