# Story E03S07: N8N — On Site Action: Customer SMS + Status Update

## User Story

**As a** customer,
**I want** to receive an SMS when the technician has arrived,
**so that** I know they're at my property and work is beginning.

---

## Context

When the tech taps On Site (E03S05), `jobs.status` updates to `On Site`. N8N sends the customer a confirmation SMS. Mirrors the En Route flow (E03S06) with a different status trigger and message.

---

## Acceptance Criteria

- [ ] **AC1:** When `jobs.status` changes to `On Site`, n8n sends an SMS to `jobs.customer_phone` within 60 seconds.
- [ ] **AC2:** SMS text: `[Business Name]: Your technician has arrived.`
- [ ] **AC3:** Business name is pulled from `clients.business_name` via `jobs.client_id` — never hardcoded.
- [ ] **AC4:** Uses n8n native Twilio node — NOT HTTP Request (per conventions).
- [ ] **AC5:** If `customer_phone` is null, the SMS step is skipped — workflow does not error.
- [ ] **AC6:** Job status in Supabase remains `On Site` regardless of SMS delivery outcome.

---

## Out of Scope

- Job Complete flow (Sprint 4)
- Email fallback for customer
- Tech check-in confirmation back to dispatcher

---

## Dependencies

- E03S05 — status change to `On Site` is the trigger
- E03S06 — same workflow pattern; build E03S07 immediately after E03S06
