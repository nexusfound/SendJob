# Story E03S03: N8N — Send Tech Notification SMS with Magic Link

## User Story

**As an** assigned technician,
**I want** to receive an SMS with my job details and a magic link when I'm assigned a job,
**so that** I can immediately see where to go and what the job is without calling anyone.

---

## Context

Once a token is generated (E03S02), n8n must send the tech an SMS containing job details and their magic link URL. This is the primary way the tech knows they've been assigned. The message uses the canonical SMS template from conventions.

---

## Acceptance Criteria

- [ ] **AC1:** Within 60 seconds of job assignment, the assigned tech receives an SMS on `techs.phone`.
- [ ] **AC2:** The SMS contains: business name, customer name, service address, customer phone, issue description, and magic link URL.
- [ ] **AC3:** Magic link URL format: `https://tech.sendjob.app?token=[token]`
- [ ] **AC4:** SMS uses the canonical template: `[Business Name] - New job assigned: [Customer], [Address], [Phone]. Issue: [Description]. View job: [URL]`
- [ ] **AC5:** Business name is pulled from `clients.business_name` — never hardcoded.
- [ ] **AC6:** If the tech's `contact_method` is `sms`, SMS is sent. If `email`, Resend email is sent instead with the same content. (Email path is acceptable as a stub — SMS is the primary path for Sprint 3.)
- [ ] **AC7:** Uses n8n native Twilio node — NOT HTTP Request (per conventions).

---

## Out of Scope

- Retry logic if SMS delivery fails
- Read receipts or delivery confirmation
- Multi-job batching (one SMS per assignment event)

---

## Dependencies

- E03S02 — token must exist before SMS can be sent

---

## Notes

- Twilio node credential type: Twilio (Account SID + Auth Token)
- Always include opt-out on first SMS to a new contact: append `Reply STOP to unsubscribe` if first job for this tech
- Business name lookup: already available in workflow context from `clients` table lookup during assignment
- n8n expression for token: `$('Generate Token').item.json.token`
- Test on an actual mobile device — field techs read these on phones, not desktops
