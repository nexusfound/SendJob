# Story E02S01: Twilio Inbound Webhook — Receive Dispatcher SMS Reply

## User Story

**As a** dispatcher,
**I want** my SMS reply to the job alert to be received and routed into the system automatically,
**so that** I can assign a technician by text without opening a browser or dashboard.

---

## Context

When a new job comes in, the dispatcher receives an SMS alert. They reply with a tech name to assign. This story connects Twilio's inbound SMS to the n8n workflow — it is the entry point for the entire SMS assignment path (E02S02–S04 depend on it).

---

## Acceptance Criteria

- [ ] **AC1:** When the dispatcher sends an SMS reply to the registered Twilio number (+13212489804), Twilio forwards the message to the n8n webhook endpoint at `https://nexusfound.cloud/webhook/sendjob-intake` (or a dedicated inbound path) over HTTPS.
- [ ] **AC2:** The n8n webhook returns a 200 OK response to Twilio within 15 seconds — preventing Twilio from triggering retry loops.
- [ ] **AC3:** The SSL certificate on `nexusfound.cloud` is valid and not expired before this is wired — verify with a browser check or `curl -I` before testing.
- [ ] **AC4:** The incoming message body (dispatcher's reply text) and sender phone number (E.164 format) are both available as fields in the n8n workflow for downstream processing.
- [ ] **AC5:** The webhook correctly receives test messages sent from the registered dispatcher phone number.
- [ ] **AC6:** If the webhook endpoint is unreachable, Twilio's retry behaviour does not cause duplicate processing — the n8n workflow handles duplicate detection or is idempotent for this entry point.

---

## Out of Scope

- Parsing the SMS content (E02S02)
- Matching tech names or updating job status (E02S02)
- Error handling for unrecognized tech names (E02S03)
- Already-assigned guard (E02S04)

---

## Dependencies

- None — can be built independently.

---

## Notes

- Twilio webhook URL points to nexusfound.cloud n8n endpoint over HTTPS — verify SSL cert is valid before wiring.
- Budget 1hr debug buffer — n8n/Twilio webhook integration is the most common snag point.
- Use Hoppscotch (hoppscotch.io) for webhook testing on Windows — no curl install required.
- Twilio sends inbound SMS as a POST with `Body`, `From`, `To` fields in `application/x-www-form-urlencoded` format.
- The registered number is +13212489804 — A2P campaign must be approved before live SMS testing.
