# Story E04S01: Tech Dashboard — Job Complete Button

## User Story

**As a** technician who has finished a job,
**I want** to tap a "Job Complete" button on my dashboard,
**so that** the job is marked complete and the customer is notified automatically without me needing to call anyone.

---

## Context

The final step in the technician's job flow. En Route and On Site buttons are already implemented (E03S05) using the same PATCH pattern. Job Complete follows identical UX — large tap target, immediate disable, inline confirmation on success.

---

## Acceptance Criteria

- [ ] **AC1:** "Job Complete" button is visible only when `jobs.status = On Site`.
- [ ] **AC2:** Button is minimum 48x48px tap target, styled consistently with En Route and On Site buttons.
- [ ] **AC3:** On tap, button is disabled immediately to prevent double-submit.
- [ ] **AC4:** PATCH `jobs` table: `status = 'Complete'`, `completed_at = new Date().toISOString()`.
- [ ] **AC5:** On success, button is replaced with a static confirmation: "Job marked complete."
- [ ] **AC6:** On failure, inline error is shown and button is re-enabled for retry. No silent failure.
- [ ] **AC7:** `completed_at` is stored in ISO 8601 format (JavaScript `new Date().toISOString()`).
- [ ] **AC8:** The job card remains visible after completion — tech sees confirmation, not a blank screen.

---

## Out of Scope

- Confirmation that the customer SMS was sent (handled by E04S02, invisible to the tech)
- Payment collection UI
- Job history / past completed jobs view

---

## Dependencies

- E03S05 — Job Complete button is added to the same `index.html` as En Route / On Site buttons. No new file.

---

## Notes

- PATCH pattern (same as E03S05):
  ```javascript
  fetch(`${SUPABASE_URL}/rest/v1/jobs?id=eq.${jobId}`, {
    method: 'PATCH',
    headers: {
      'apikey': SUPABASE_ANON_KEY,
      'Authorization': 'Bearer ' + SUPABASE_ANON_KEY,
      'Content-Type': 'application/json',
      'Prefer': 'return=representation'
    },
    body: JSON.stringify({ status: 'Complete', completed_at: new Date().toISOString() })
  })
  ```
- Anon RLS UPDATE policy already covers `jobs` table — no schema change needed for this story
- `completed_at` column already exists in schema (nullable timestamp) — no migration needed
- After marking complete, the real-time subscription on the dispatcher dashboard will update automatically
