# Story E03S04: Technician Dashboard â€” Job List View (Magic Link Auth)

## User Story

**As a** technician,
**I want** to open my magic link and immediately see all my active jobs with key details,
**so that** I know where to go and what to do without needing to log in or call the office.

---

## Context

The tech dashboard is a single `index.html` deployed to Netlify at `tech.sendjob.app`. It reads the `?token=` query parameter, validates it against `techs.magic_link_token` in Supabase, and displays all active jobs assigned to that tech. No login screen â€” the token IS the auth. Must be designed for one-handed use on a phone in a truck.

---

## Acceptance Criteria

- [ ] **AC1:** Page reads `?token=` from the URL query string on load.
- [ ] **AC2:** Token is validated by querying `techs` table: `GET /rest/v1/techs?magic_link_token=eq.[token]&select=id,name,client_id`. If no match, redirect to error state (E03S08).
- [ ] **AC3:** On valid token, all jobs with `tech_id = techs.id` AND `status` IN (`Assigned`, `En Route`, `On Site`) are fetched and displayed.
- [ ] **AC4:** Each job card displays: Job ID (last 6 chars), Customer Name, Service Address, Service Type, Issue Description, Current Status.
- [ ] **AC5:** Jobs are ordered by `assigned_at` ascending â€” oldest assignment first.
- [ ] **AC6:** Dashboard is optimized for mobile â€” minimum tap target 48px, readable at arm's length, no horizontal scroll on 375px viewport.
- [ ] **AC7:** If no active jobs exist for this tech, a friendly empty state is shown: "No active jobs right now."
- [ ] **AC8:** Page does not show a login screen â€” token in URL is the only auth mechanism.
