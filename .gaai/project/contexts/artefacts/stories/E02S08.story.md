# Story E02S08: Dispatcher Login — Supabase Auth (Email/Password)

## User Story

**As a** dispatcher,
**I want** to log in with my email and password before accessing the job queue,
**so that** job data is protected from unauthorized access.

---

## Context

The dashboard is currently open to anyone with the URL. Before it goes live, it must require authentication. Supabase Auth (email/password) is the confirmed auth method for dispatchers. The login must gate the entire dashboard — unauthenticated users see only the login form.

---

## Acceptance Criteria

- [ ] **AC1:** When an unauthenticated user visits the dashboard URL, they see a login form — not the job table.
- [ ] **AC2:** The login form collects email and password. On submit, it authenticates via the Supabase Auth JS client (`supabase.auth.signInWithPassword`).
- [ ] **AC3:** On successful login, the job queue loads and the login form is hidden.
- [ ] **AC4:** On failed login (wrong credentials), a clear error message is shown — e.g., `"Invalid email or password."` — without clearing the email field.
- [ ] **AC5:** The authenticated session persists across page refreshes within the same browser session — the dispatcher does not need to log in again on refresh.
- [ ] **AC6:** A visible logout option is available. Clicking it signs the dispatcher out and returns them to the login form.
- [ ] **AC7:** The login form is fully usable on mobile (375px) — email and password inputs meet `16px` font size (prevents iOS zoom) and `52px` min-height tap targets.
- [ ] **AC8:** After login, RLS policies on the `jobs` table correctly allow the authenticated dispatcher to read and update job records.

---

## Out of Scope

- Forgot password / password reset flow
- Multi-dispatcher accounts or role-based access
- Social login (Google, GitHub, etc.)
- Tech authentication (magic link — Sprint 3)

---

## Dependencies

- E02S05 — dashboard HTML must exist to add the auth gate to.

---

## Notes

- Use Supabase Auth JS client (`@supabase/supabase-js` or CDN) — inline in the single `index.html`.
- Supabase CDN: `https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2/dist/umd/supabase.min.js`
- Auth flow: on page load, call `supabase.auth.getSession()` — if no session, show login form; if session exists, show job queue.
- RLS policies on `jobs` table are already set (anon SELECT/INSERT/UPDATE from Sprint 1). Confirm these also work for authenticated users — may need to add `auth.role() = 'authenticated'` policies.
- Design system: login form uses `--navy-mid` background, `--border` borders, `--ice` focus ring, Barlow font, `16px` input font size — match the Playbook design.
- Test on both desktop and mobile before marking done.
- Dispatcher account must be created in Supabase Auth (Dashboard → Authentication → Users) before testing.
