# Story E02S06: Dashboard — Urgent Jobs Visually Flagged

## User Story

**As a** dispatcher,
**I want** urgent jobs to stand out visually in the job list,
**so that** I can identify and prioritize them instantly without reading every row.

---

## Context

The job list already shows an URGENT/Normal badge in the Urgency column (E02S05). However, on a busy queue — especially on mobile — a single column badge is easy to miss. This story adds a row-level visual treatment so urgent jobs are unmistakable at a glance.

---

## Acceptance Criteria

- [ ] **AC1:** Jobs where `is_urgent = true` have a distinct row-level visual treatment — a red left border, red row background tint, or equivalent — using the design system's `--urgent` (`#FF4444`) and `--urgent-glow` (`rgba(255,68,68,0.15)`) colors.
- [ ] **AC2:** The urgent row treatment is immediately visible on a 375px mobile screen without needing to locate the Urgency column.
- [ ] **AC3:** Non-urgent rows are visually unaffected by this change.
- [ ] **AC4:** The treatment works correctly in both the desktop table view and the mobile card stack view.
- [ ] **AC5:** The existing URGENT badge in the Urgency column (from E02S05) is retained — this story adds to it, not replaces it.

---

## Out of Scope

- Sound or push notification for new urgent jobs
- Sorting urgent jobs to the top of the list (sort order remains oldest-first per E02S05)
- Any animation beyond what the design system provides

---

## Dependencies

- E02S05 — dashboard must exist with the job table rendered.

---

## Notes

- `is_urgent` boolean is already fetched in the E02S05 query — no additional Supabase call needed.
- Design system colors to use: `--urgent: #FF4444`, `--urgent-glow: rgba(255,68,68,0.15)`.
- Suggested implementation: red left border (`border-left: 3px solid var(--urgent)`) + subtle background tint on the `<tr>`.
- Mobile card view: the card `border` should change to `border-color: var(--urgent)` for urgent jobs.
- This is a CSS-only change — no JS logic changes required beyond adding a conditional class on the `<tr>`.
