# Story E02S03: n8n — Error SMS if Tech Name Not Recognized

## User Story

**As a** dispatcher,
**I want** to receive an error SMS when my reply doesn't match a known tech name,
**so that** I can immediately correct my reply without having to look up the tech roster elsewhere.

---

## Context

When the dispatcher's SMS reply doesn't match any tech in the client's roster (E02S02 finds no match), the system must recover gracefully. Leaving the dispatcher with no response would stall the assignment flow. This story adds the error branch.

---

## Acceptance Criteria

- [ ] **AC1:** When n8n finds no matching tech for the dispatcher's SMS reply, an error SMS is sent back to the dispatcher's phone number.
- [ ] **AC2:** The error SMS clearly states no match was found and lists the valid tech names/codes for that client — e.g., `"No match for '[reply]'. Valid techs: Mike, Jake, Sara. Reply with a name to assign."`
- [ ] **AC3:** The error SMS is sent from the same Twilio number the dispatcher originally received the job alert from (matched via `clients.twilio_number`).
- [ ] **AC4:** The job status remains unchanged — still `New`, no partial assignment.
- [ ] **AC5:** The valid tech name list is pulled live from the `techs` table at the time of the error — not hardcoded.

---

## Out of Scope

- Already-assigned guard (E02S04)
- Fuzzy matching or autocorrect of tech names

---

## Dependencies

- E02S02 — the tech lookup logic must exist for this branch to activate.

---

## Notes

- This is an `if no match` branch in the n8n workflow after the tech lookup in E02S02.
- Fetch all techs for the client: `GET /rest/v1/techs?client_id=eq.{client_id}&select=name` — build the list from the response.
- Use native n8n Twilio node to send the error SMS (NOT HTTP Request).
- Keep the error message short — SMS character limit is 160 chars for single segment.
