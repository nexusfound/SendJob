# Story E03S02: N8N — Generate Magic Link Token on Job Assignment

## User Story

**As a** dispatcher,
**I want** the system to automatically generate a secure magic link for the assigned technician when a job is assigned,
**so that** the tech can access their job details without needing a login account.

---

## Context

When a job is assigned (either via SMS reply or dashboard dropdown), n8n must generate a unique token, store it in the `techs` table, and make it available for the subsequent tech SMS notification (E03S03). The token is scoped per-tech (not per-job) — one active token shows all jobs assigned to that tech.

---

## Acceptance Criteria

- [ ] **AC1:** When `jobs.status` is updated to `Assigned`, n8n generates a cryptographically random token (UUID or equivalent) for the assigned tech.
- [ ] **AC2:** The token is stored in `techs.magic_link_token` and `techs.token_issued_at` (current timestamp) via Supabase HTTP Request PATCH.
- [ ] **AC3:** A new assignment overwrites the previous token for that tech — only one active token per tech at any time.
- [ ] **AC4:** Token generation is triggered by both assignment paths: SMS reply (E02S02) and dashboard dropdown (E02S07).
- [ ] **AC5:** If the Supabase PATCH fails, n8n logs the error — the job remains Assigned (token failure must not roll back the assignment).

---

## Out of Scope

- Token expiry enforcement (covered by E03S08 and Sprint 4)
- Sending the SMS (covered by E03S03)
- Token validation on the tech dashboard (covered by E03S04)

---

## Dependencies

- E02S02 (SMS assignment path) and E02S07 (dashboard assignment path) — both must trigger this flow

---

## Notes

- Use n8n Code node to generate token: `return [{ json: { token: require('crypto').randomUUID() } }]`
- PATCH to `techs` table: `PATCH /rest/v1/techs?id=eq.[tech_id]` with `{ magic_link_token: token, token_issued_at: now() }`
- Use HTTP Request node — NOT native Supabase node (DEC-003)
- Add `Prefer: return=representation` header to confirm write succeeded
- n8n field reference pattern: `$('Node Name').item.json.field` — never `$json.field` for upstream nodes
