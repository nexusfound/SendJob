# Build Skills Index

## Purpose / When to Activate

Activate when:
- A new skill is created (invoked automatically by `create-skill` Step 6)
- A skill's frontmatter is modified (description, tags, category, id)
- A skill is removed or deprecated
- `skills-index.yaml` is absent or suspected stale (e.g. after a git pull or framework update)

This skill generates the derived catalog used by agents for fast skill discovery.
It does not replace frontmatter — it aggregates it.

**`skills-index.yaml` is a cache. Frontmatter is the source of truth.**

---

## Process

### Step 1 — Traverse the skills directory

Scan `.gaai/core/skills/` recursively. Collect every file named `SKILL.md`.
Ignore non-SKILL.md files. Ignore `README.skills.md`, `skills-index.yaml`.

### Step 2 — Extract frontmatter from each SKILL.md

For each `SKILL.md` found, read the YAML frontmatter block (between `---` delimiters).
Extract the following fields:

| Field | Source | Notes |
|---|---|---|
| `name` | frontmatter `name` | Required |
| `description` | frontmatter `description` | Required |
| `id` | frontmatter `metadata.id` | Required |
| `category` | frontmatter `metadata.category` | Required |
| `track` | frontmatter `metadata.track` | Required |
| `tags` | frontmatter `metadata.tags` | May be absent — default to `[]` |
| `version` | frontmatter `metadata.version` | May be absent |
| `updated_at` | frontmatter `metadata.updated_at` | May be absent |
| `path` | derived — relative path from `.gaai/core/skills/` to SKILL.md | e.g. `cross/memory-ingest/SKILL.md` |

If a required field is missing, log a warning inline in the entry (`_warning: "missing field: name"`) but do not skip the entry — include it for visibility.

### Step 3 — Group entries by category

Organize entries into three groups: `discovery`, `delivery`, `cross`.
Within each group, sort alphabetically by `name`.

### Step 4 — Write skills-index.yaml

Write to `.gaai/core/skills/skills-index.yaml` using the format below.
Overwrite any existing file — this is a fully derived artifact.

### Step 5 — Report

Return to the invoking agent:
- Total skills scanned
- Any entries with missing required fields (names + fields)
- Any entries with duplicate `id` values

## Quality Checks

- Every SKILL.md in the directory is represented in the output (no silently skipped files)
- No manually written entries — all entries derived from frontmatter
- Duplicate `id` values flagged (do not silently deduplicate)
- `generated_at` reflects the date of this run
- File written to `.gaai/core/skills/skills-index.yaml` exactly (not a variant path)

---

## Non-Goals

This skill must NOT:
- Edit any SKILL.md file
- Make decisions about which skills are valid or relevant
- Merge duplicate skills or resolve conflicts — only report them
- Be invoked as a dependency of other skills (only agents call this)

**This skill reads and aggregates — it does not evaluate or decide.**
